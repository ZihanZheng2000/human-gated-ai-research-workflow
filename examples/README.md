# Examples

Public examples should be small, inspectable, and organized according to `../docs/example-organization.md`.

The initial public release keeps this folder lightweight. Larger exploratory runs should be cleaned before publication so that private notes, large generated artifacts, copyrighted materials, and environment-specific files are not uploaded accidentally.

Recommended future examples:

- `framework-smoke-test-public-data/`: a small synthetic-gate public-data smoke test.
- `water-environment-research-planning/`: a domain-owned Planning case.
- `cross-domain-evaluation/`: future expert-gated Education, Health, and Energy/Climate case studies.
